# Web Modules Structure

Create module folders using this pattern:

- `appointments/`
- `patients/`
- `billing/`
- `consultations/`
- `diagnostics/`
- `pharmacy/`
- `emr/`
- `notifications/`image.png

Recommended internal layout per module:

- `api/` for backend calls
- `components/` for reusable module UI
- `hooks/` for feature state management
- `pages/` for route-level screens
- `schemas/` for validations
- `types/` for contracts
